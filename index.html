sleuth输出的日志格式

可以看到内容是由[appname,traceId,spanId,exportable]组成的, 具体含义如下：

  • appname：服务的名称，也就是spring.application.name的值
  • traceId：整个请求的唯一ID，它标识整个整个请求的链路
  • spanId：基本的工作单元，发起一起远程调用就是一个span
  • exportable：是否导入数据到Zipkin中

线程池类型

dubbo 提供了4中线程池类型

  • fixed

    固定大小线程池，启动时建立线程，不关闭，一直持有(缺省)

    • core-size

      默认值：200，可配置

    • max-size

      默认值：200，可配置

    • queue-size

      默认值：0，可配置

      • 等于 0 时：SynchronousQueue
      • 不等于 0 时：LinkedBlockingQueue

    • keep-alive

      默认值：0
